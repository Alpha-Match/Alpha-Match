# `CategoryPieChart` 레이블 가시성 개선

**날짜**: 2026-01-06
**작성자**: Gemini Pro
**목적**: `CategoryPieChart`에서 점유율이 낮은 파이 조각의 레이블이 표시되지 않던 문제를 해결하여 모든 카테고리 정보가 차트 내에 명확하게 나타나도록 합니다.

---

## 📋 작업 요약

이전에 `CategoryPieChart.tsx`에 구현된 `CustomPieLabel` 컴포넌트에는 점유율이 5% 미만인 파이 조각의 레이블을 생략하는 로직이 있었습니다. 이로 인해 'Collaboration / Project Management'와 같이 점유율이 낮은 카테고리가 차트 내에서 전혀 표시되지 않아 사용자가 모든 정보를 파악하기 어렵다는 피드백이 있었습니다. 본 작업을 통해 이 문제를 해결하여 모든 카테고리의 레이블이 차트 내에 표시되도록 개선했습니다.

---

## ✅ 완료된 작업 상세 내역

### 1. `CategoryPieChart.tsx` 레이블 가시성 로직 수정

#### 문제점
-   `CustomPieLabel` 컴포넌트 내 `if (percent < 0.05)` 조건으로 인해 점유율이 5% 미만인 파이 조각의 레이블이 렌더링되지 않았습니다.

#### 해결 방안
1.  **조건부 렌더링 로직 제거**: `CustomPieLabel` 컴포넌트에서 `if (percent < 0.05) return null;` 코드를 제거했습니다. 이로써 모든 파이 조각에 대해 레이블 렌더링이 시도됩니다.
2.  **레이블 색상 조정**: `fill="white"`로 고정되어 있던 레이블 텍스트의 색상을 Tailwind CSS의 `--color-text-primary` 변수를 참조하도록 `fill="var(--color-text-primary)"`로 변경하여, 현재 테마의 기본 텍스트 색상을 따르도록 개선했습니다.

---

## 📊 UI/UX 개선 효과

-   **정보 접근성 향상**: 점유율과 관계없이 모든 카테고리의 이름과 백분율이 파이 차트 내에 직접 표시되어, 사용자가 범례를 참조할 필요 없이 모든 정보를 한눈에 파악할 수 있게 되었습니다.
-   **시각적 명확성 증가**: 모든 카테고리에 대한 정보가 차트 자체에 제공되어 데이터 해석의 직관성과 효율성이 높아졌습니다.
-   **피드백 반영**: 사용자 피드백을 직접 반영하여 UI가 개선되었습니다.
